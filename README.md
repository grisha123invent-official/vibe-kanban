<h1 align="center">Vibe Kanban (Local-First Agent Orchestrator)</h1>

<p align="center">Получите в 10 раз больше пользы от Claude Code, Gemini CLI, локальных LLM и других ИИ-агентов...</p>

## Что это такое?

В мире, где инженеры тратят большую часть времени на планирование и ревью кода, написанного ИИ-агентами, наиболее эффективный способ повысить продуктивность — ускорить процесс планирования работы и проверки результатов.

**Vibe Kanban** создан именно для этого. Используйте классические kanban-доски для планирования задач. А когда будете готовы приступить к работе, доверьте выполнение ИИ-агентам прямо в вашем локальном репозитории.

**Особенности версии Local-First:**
- **Локальные директории вместо облачных проектов** — просто выберите папку на вашем компьютере, и она мгновенно станет проектом на доске.
- **Поддержка любых API (включая Kia / OpenRouter)** — указывайте кастомные URL агрегаторов и ключи для минимизации затрат на ИИ.
- **Глобальная Канбан-доска** — агрегация всех задач из всех ваших локальных папок на одном экране.
- **Свои PM-Оркестраторы** — управляющий агент может сам подхватывать ваши локальные `@skills`, планировать работу и раскидывать подзадачи другим агентам.
- **Поддержка 10+ ИИ-исполнителей** — переключайтесь между Claude Code, Gemini CLI, локальными моделями Ollama, GitHub Copilot, Cursor и другими.

## Установка и запуск

Поскольку мы используем полностью локальную отвязанную от облака версию, для запуска сервера разработки выполните в терминале в корне проекта:

```bash
pnpm i
pnpm run dev
```

Откроется дашборд (обычно на порту `3001` или `3000`), где вы сразу сможете начать добавлять локальные папки в качестве проектов.

## Разработка

### Требования

- [Rust](https://rustup.rs/) (последняя стабильная версия)
- [Node.js](https://nodejs.org/) (>=20)
- [pnpm](https://pnpm.io/) (>=8)

### Сборка Frontend-части отдельно

Если вам нужно собрать только React-часть:

```bash
cd packages/local-web
pnpm run build
```
